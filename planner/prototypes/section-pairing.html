Section-pairing prototypes

Same scenario in four shapes. The outer Kanban column is unchanged — what differs is how a single column expresses section-level nesting (e.g. "Lec 001 paired with Lab 003" vs "Lec 002 paired with Lab 005"). Pick which feels right to live inside one Kanban column.

Scenario
Course = ENGR-UH 1000. The user wants either pairing (Lec 001 AND Lab 003) or (Lec 002 AND Lab 005). Other combinations (e.g. Lec 001 + Lab 005) are not acceptable.
A · Combo cards Each pairing = one stacked card

A combo card lists every component of one valid pairing, stacked with thin dividers. Multiple cards stack as alternatives, separated by italic or. New combo via "+ pairing" button opens a builder.

Strengths: the unit of meaning is "one valid schedule slice" — easy to read; matches how students think ("either this combo or that combo"); compact.
Risks: editing inside a card is awkward (modal? inline?); duplicates verbose if many combos share a lecture; doesn't naturally express "any lab works" inside a card.
B · Pairings grid Two-axis matrix: components × picks

Rows = component types (Lec, Lab, Rec). Columns = each acceptable pairing. Each cell holds the chosen section for that component-in-that-pairing. Add a new column to add another pairing.

C · Mini tree Recursive AND/OR boxes, scoped to one column

The same nested-boxes UI from earlier, miniaturized and confined to one Kanban column. Top-level group is or; each child is either a single section or an and-group of sections that must all be picked together.

Strengths: arbitrary expressions; consistent mental model with the earlier prototype; flexible for edge cases ("(A and B) or (C and any lab)").
Risks: visually busier per column; the very thing we just stepped back from. Best if section-pairing is rare and isolated — Kanban for the common case, mini-tree for the one weird course.
D · Stitched chips Drag pills together; fused = AND, stack = OR

Each pill is one section. Drag a pill onto another to fuse them — they become one "pair card" joined by +. Stack pair cards vertically to express alternatives. Single unfused pills = "any pairing that includes this section is OK".

Pick one of sections
ENGR-UH 1000

or
Lec § 002 + Lab § 005
drop a section here
Strengths: tactile metaphor (literally stick sections together); compact horizontal pairs; visual language is different from the outer Kanban so users don't conflate scopes.
Risks: drag-onto-pill may not be obvious (needs onboarding); harder to express deeper nesting (3+ components in a pair gets crowded).

My pick: A (Combo cards) for clarity, with D (Stitched chips) as a close second if you want a more playful interaction model. C is most powerful but visually heavy; B is rigid and the "any" semantics feel awkward in practice.
